# ADR-0007: Markdownダウンロード機能とページ複製機能を実装しない

## ステータス

決定済み (2026-02-09)

## コンテキスト

以下の2つの機能提案があった:

- **Issue #109: Markdownファイルのダウンロード機能** — エディタページにダウンロードアイコンを追加し、ページの内容を `.md` ファイルとしてエクスポートする機能。ブラウザAPIのみで実装可能。
- **Issue #112: ページの複製機能** — ページ一覧にコピーボタンを追加し、既存ページの内容をそのまま新しいページに複製する機能。APIの拡張を伴う。

どちらも「あったら便利かもしれない」機能であり、既存の操作（コピー&ペースト）で同じ目的を達成できる。

## 決定

Issue #109（Markdownダウンロード機能）と Issue #112（ページ複製機能）はいずれも実装しない。

## 理由

### コピー&ペーストで代替可能

どちらの機能も、ユーザーがエディタ上でテキストを全選択してコピーし、ファイルや新しいページにペーストすることで目的を達成できる。これはテキストエディタにおいて広く認知された操作であり、ユーザーに追加の学習を強いることはない。

- ダウンロード: 内容をコピーし、任意のテキストエディタに貼り付けて保存する
- 複製: 内容をコピーし、新しいページを作成して貼り付ける

専用機能を追加しなければ達成できない体験の向上が存在しない。

### UIの複雑化

ダウンロードボタンや複製ボタンを追加すると、エディタやページ一覧のUIにアイコンが増える。一つ一つは小さな追加であっても、これらが積み重なることでインターフェースのノイズとなり、ミニマルな体験を損なう。ホワイトボードや紙のノートには「ダウンロード」ボタンは存在しない。

### ミニマリズムの維持

このプロダクトの価値は「余計なものがない」ことにある。「あったら便利かもしれない」という基準で機能を追加し始めると、際限なくUIが肥大化する。プロダクト哲学の「機能は必要最小限に。『あったら便利かも』は追加しない」という原則に従い、明確に追加しないと判断する。

### 判断基準の適用

- **それがなくても目的は達成できないか？** → コピー&ペーストで達成できる。専用機能がなくても困らない
- **説明なしで使い方がわかるか？** → ダウンロードアイコンや複製ボタンは直感的ではあるが、そもそも提供する必要がない機能に対してこの基準を適用する意味がない
- **ユーザーの思考を中断させないか？** → ボタンの追加自体が視覚的ノイズとなり、書くことに集中する体験を妨げる可能性がある

## 影響

- Issue #109 および Issue #112 はクローズする
- エディタページおよびページ一覧のUIは現状を維持する
- 今後、同様の「便利機能」の提案があった場合、本ADRを判断の参考とする。コピー&ペーストなどの汎用操作で代替可能な機能は、専用UIを追加する十分な理由がない限り実装しない
