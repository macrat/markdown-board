# ADR-0009: モバイルでの空白ページ自動フォーカスを抑制しない

## ステータス

決定済み (2026-02-12)

## コンテキスト

Issue #182 にて、モバイルデバイスで空白ページを開いた際の自動フォーカスを抑制する提案があった。自動フォーカスによりソフトキーボードが画面の約半分を覆うため、モバイルではフォーカスをスキップし、タップで編集を開始する方式への変更が提案された。

## 決定

モバイルデバイスでの自動フォーカス抑制は実装しない。現在の挙動（全デバイスで空白ページに自動フォーカス）を維持する。

## 理由

### モバイルデバイスこそ自動フォーカスが必要

モバイルデバイスではソフトキーボードを表示するために画面をタップする操作が必要であり、デスクトップに比べて入力開始までの手間が大きい。外出先で素早くメモを取りたい場面こそ、開いた瞬間から書き始められる即時性が重要である。自動フォーカスを抑制すると、この即時性が失われる。

### プロダクトの核心的価値との一致

「開いた瞬間から書き始められる」ことはこのプロダクトの5つのコア原則のうち「即座に始められる」に直結する。ソフトキーボードが表示されることは「入力の準備が整っている」状態を意味し、ユーザーの意図（書くこと）に沿った振る舞いである。

### キーボード表示は問題ではなく機能

ソフトキーボードが画面を覆うことを「問題」と捉える視点もあるが、空白ページを開くユーザーの目的は書くことである。キーボードが表示されていることは、その目的に即した状態であり、むしろ余計な操作（タップしてキーボードを出す）を省いている。

## 影響

- Issue #182 はクローズする
- 空白ページの自動フォーカスは全デバイスで維持する
- 今後、モバイル向けに操作を増やす（タップを要求する）提案があった場合、本ADRを参考に、即時性を損なわないか慎重に判断する
