# Markdown Board

開いた瞬間から書き始められる、ミニマルな共同編集メモツール。

## プロダクトコンセプト

### ビジョン

**「悩む」から「考える」へ**

Markdown Board は、紙のノートやホワイトボードのように「深く考えずに様々なことを書き出せる」ミニマルなメモツールです。頭の中のモヤモヤを書き出すことで、「悩んでいる」状態から「考えている」状態へと変化する——その瞬間を生み出すことがこのプロダクトの価値です。

### コア哲学

#### 1. ノイズの排除
ユーザーの思考を妨げるあらゆるノイズを排除します：
- **UIの複雑さ**: ボタンやメニューは最小限に
- **保存・同期の意識**: 自動保存、シームレスな同期
- **フォーマットの強制**: 構造を強いらない、自由に書ける

#### 2. 即座に始められる
開いた瞬間から書き始められることが最重要です。学習コストをゼロに近づけ、何も考えずに使い始められる直感性を追求します。

#### 3. ミニマリズム
機能は必要最小限に。「あったら便利かも」は追加しない。本当に必要な機能だけを、美しいUIで提供します。

#### 4. リアルタイムの共有体験
複数人が同時に編集できることで、ホワイトボードを囲んでいるような体験を提供します。固定チームではなく、その場その場で柔軟にコラボレーションできることを重視します。

#### 5. ストレスなく使える
すべての操作は、ユーザーに違和感を与えない自然な挙動で動作します。「遅い」と感じさせないスピードを保ち、待ち時間によって思考が中断されることを防ぎます。

### Markdownを採用する理由

- **構造と自由のバランス**: 最低限の構造を提供しつつ、自由に書ける
- **軽量さ**: テキストベースでデータが軽く、扱いやすい
- **アクセシビリティ**: 知らなくても普通に使える、知っていればより便利に

### UIデザイン原則

- **余白と静謐**: 空白を大切にし、静かで落ち着いた印象
- **機能美**: 装飾ではなく、機能そのものが生み出す美しさ

### ターゲットユーザー

- **メイン**: 個人での思考整理やメモ
- **サブ**: 2〜5人程度の柔軟なコラボレーション（固定チームより、その都度の共同作業）

## 開発ガイドライン

### やるべきこと

- 開いた瞬間から使える即時性を守る
- 保存や同期をユーザーに意識させない設計
- 機能追加の前に「本当に必要か」を問う
- シンプルさを維持するためのコードレビュー
- ユーザーの思考の流れを妨げないUX

### やってはいけないこと

- **機能の肥大化**: 「便利そう」という理由で機能を追加しない
- **カスタマイズ設定の追加**: ユーザーに選択を強いない
- **ワークフローの強制**: 特定の使い方を押し付けない
- **複雑な整理機能**: シンプルさを損なう階層化やタグ付けの過剰導入
- **学習を要するUI**: 説明が必要なインターフェースは設計の失敗

### 判断に迷ったら

機能追加や変更を検討するとき、次の問いを自分に投げかけてください：

1. **それがなくても目的は達成できないか？** — 達成できるなら、追加しない
2. **説明なしで使い方がわかるか？** — わからないなら、設計を見直す
3. **ユーザーの思考を中断させないか？** — 中断させるなら、やり方を変える

迷いが解消しないときは、追加しない方を選んでください。後から足すことはできても、一度増えた複雑さを取り除くのは難しいからです。

## セットアップ

```bash
git clone https://github.com/macrat/markdown-board.git
cd markdown-board
npm install
npm run dev
```

http://localhost:3000 を開く。

### 環境変数

プロジェクトでは以下の環境変数を使用できます。必要に応じて `.env.local` ファイルを作成して設定してください。

| 変数名 | 説明 | デフォルト値 |
|--------|------|--------------|
| `NEXT_PUBLIC_WS_PORT` | クライアント側で使用するWebSocketサーバーのポート番号 | `1234` |
| `API_URL` | クリーンアップスクリプト（`scripts/cleanup-archives.js`）で使用するAPIサーバーのベースURL | `http://localhost:3000` |

詳細は `.env.example` ファイルを参照してください。
